# 先写文档

文档就 md 写在项目根目录下 我的就叫 PROJECT.md , 在 ClAUDE.md 里面引用

不要觉得这麻烦

最开始我先多和 GPT 5.2 探讨需求和功能，以及竞品分析
我认为 GPT 5.2 在这方面的能力很棒

主要技术探讨我会和 Claude 4.6 Opus 探讨

1. 当要做一个全新项目时候，应该先做好需求分析和功能规划，明确项目的核心价值和用户痛点。
2. 技术选型时要考虑项目的长期维护性和扩展性
3. 关于技术选型, 我是全 ts ，ai 写 ts 太强了，强烈建议选主流的技术 ，AI 生成的准确度高
编程语言上 脚本语言 ts py > 静态语言 cpp rust

让 ai 多查文档

方向错了，努力白费
一定要把方向选对，选好，后面节省非常多时间
核心功能一定在编码前，做好调研分析，选好包
可以多用 ai 对比下 几种技术方案的优缺点

需求拆分小点

claude 4.6 opus 能力太炸裂了，基本上都是一遍过

做好 format lint check 
在 claude.md 里要求规范，每次完成一个任务，就要做 format lint check
极大提升 vibe coding的质量

如果你要做多端应用

先做好 web 端， 然后让 ai 参考 做 mobile 和 extension 就很简单了
比如 `/plan 参考我 web 中的xx，在 native 中实现` 
web 端也方便调试
